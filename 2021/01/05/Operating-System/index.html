连续内存分配
固定分区
def： 操作系统提前按固定大小(可相等，也可不相等)，分配好区域
每个分区一个时间内只能分配给一个作业
作业装入分区：找到能容纳作业的最小的分区
// 内碎片：装入作业，分区有空闲空间无法利用
缺点：
分区大小固定
小作业可能大空间，大作业可能没空间
分区个数固定
限制了并发程度
动态分区
def：实现不确定分区大小，而是作业到，根据作业大小确定分区大小
// 外碎片：分区之间的空隙，由于太小无法分配给作业，产生外碎片
分配算法
首次适配
找到第一个可以装入的位置
最佳适配
找到最小的空闲位置装入
// 但是外碎片最多
最差适配
找到最大的空闲位置装入
// 外碎片最少
// 小结
整个作业都存储在连续空间中
容易实现地址变换
无法实现虚拟存储技术
无法实现内存共享
容易产生碎片
支持多道程序
大作业不易找到合适的分区

主要思想：
把具有逻辑意义的部分划分成一个段，以段作为单位进行存储管理而不是作业

流程
1）比较段号s和段表长度寄存器，判断是否超出段长
2）使用段号s在段表中查找段基址和段长
3）比较段内偏移量是否小于段长
