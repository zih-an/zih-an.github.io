## 内存管理
连续内存分配
    固定分区
        def： 操作系统提前按固定大小(可相等，也可不相等)，分配好区域
        每个分区一个时间内只能分配给一个作业
        作业装入分区：找到能容纳作业的最小的分区
        // 内碎片：装入作业，分区有空闲空间无法利用
        缺点：
            分区大小固定
                小作业可能大空间，大作业可能没空间
            分区个数固定
                限制了并发程度
    动态分区
        def：实现不确定分区大小，而是作业到，根据作业大小确定分区大小
        // 外碎片：分区之间的空隙，由于太小无法分配给作业，产生外碎片
        分配算法
            首次适配
                找到第一个可以装入的位置
            最佳适配
                找到最小的空闲位置装入
                // 但是外碎片最多
            最差适配
                找到最大的空闲位置装入
                // 外碎片最少
    // 小结
        整个作业都存储在连续空间中
        容易实现地址变换
        无法实现虚拟存储技术
        无法实现内存共享
        容易产生碎片
        支持多道程序
        大作业不易找到合适的分区
分段
    主要思想：
        把具有逻辑意义的部分划分成一个段，以段作为单位进行存储管理而不是作业
    段表
        def. 记录每个段在内存中的位置
        保存在内存中
            格式：段号 | 段基址 | 段长
        硬件支持：
            段基址寄存器
            段长寄存器
    地址变换
        MMU(存储器管理单元)
            负责将逻辑地址转换为物理地址
            2个寄存器
                段表基址寄存器
                段表长度寄存器
        流程
            1）比较段号s和段表长度寄存器，判断是否超出段长
            2）使用段号s在段表中查找段基址和段长
            3）比较段内偏移量是否小于段长
            4）物理地址=段基址+偏移量
    分段特点
        由程序员划分
        具有逻辑意义
        段大小不等
        虚拟地址格式：段号 + 段内位移
        作业在内存中不必连续存放
        每个段仍然连续分配，碎片问题仍然存在
分页
    主要思想：（集装箱式）均分作业大小为几“页”，同样按相同的大小将物理地址空间分为几个“页框”，这样“页”就可以分散地放置在离散的不同“页框”中
        分页图示
    页表
        格式：页号 | 页框号
        // 物理地址 = 页框号+页内偏移量
        为了降低时间开销，引入TLB快表：
            在CPU中单独设置的存储区
            仅存部分页表
            按关键字查找
            并行查找技术
            基于局部性原理
    (分段与分页比较)特点：
        不同点：
            页是操作系统分的；段是程序员分的
            页的大小是2^n且都相等；段的大小不等
            页不具有逻辑意义；段具有逻辑意义
            页在宏观上没有碎片；段仍然有碎片
        相同点：
            都是不连续存放的
            地址变换过程类似
    页的共享
        每个作业一个页表，不同作业共享相同的页框（指向相同的页框号）
    多级页表
        目的：为了解决地址空间增大，导致页表增大
        
    段页式
        def. 先分段再分页
        虚拟地址：段号 | 页号 | 页内偏移量
        
请求调页
    def. 不在一开始全部调页进内存，而是在需要访问该页部分的内容，但是不在内存中，调页
    内存映射文件：实现了像访问内存那样处理文件的输出/输出操作
        采用请求调页技术，将要访问的磁盘块装入到内存的一个页框中
        后续读、写该磁盘块时，直接访问内存中对应的页框
缺页率
    局部性原理
        时间局部性
        空间局部性
    局部：
        进程访问的页的集合，这些页在使用上具有关联性
    局部模式：
        进程的执行过程中，其访问的内存空间从一个局部迁移到另一个局部，这种运行方式被称为局部模式
    抖动(或称为“颠簸”)：
        def. 频繁的页面调度行为
        CPU利用率低，缺页率高
        引入更多的进程，缺页率进一步升高
    降低缺页率的方法：
        分配足够的内存（页框数量）
            def. 足够容纳进程当前局部
            页框分配算法
            子主题 3
        置换算法
            def. 将进程局部的页留在内存，其他的置换出去
            置换策略与抖动的关系
                可消除抖动
                也可能使抖动蔓延
分配页框
    固定分配方法
    平均分配
    按比例分配
    按优先级分配
    可变分配方法
    全局置换
    局部置换
置换算法
    def. 选择要淘汰的页，调入新页
    先进先出FIFO
    最久未用置换算法
    CLOCK置换算法
